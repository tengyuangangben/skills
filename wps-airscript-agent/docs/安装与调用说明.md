## 二、首次调用前必须配置

### 1) 环境变量（全局一次）

- `WPS_AIRSCRIPT_TOKEN`：WPS 脚本令牌（所有表通用）
- 可选 `WPS_WEBHOOK_MAP_PATH`：路由配置文件路径，不填时默认读取项目内 `wps_webhook_map.json`
- 可选 `WPS_SUBMITTER`：默认提交人（不传时默认 `agent`）
- 可选 `WPS_SUBMIT_CHANNEL`：默认提交渠道（例如 `telegram` / `feishu` / `whatsapp`）

#### 1.1) 如何获取 `WPS_AIRSCRIPT_TOKEN`

1. 打开任意一个已创建的 WPS 多维表脚本（录入/查询/字段查询都可以）。
2. 在脚本编辑器右上或工具栏点击“脚本令牌”。
3. 在弹窗中点击创建（若已存在可直接复制）。
4. 复制令牌字符串，写入本地环境变量 `WPS_AIRSCRIPT_TOKEN`。
5. 若令牌有过期时间，临近过期请在同一弹窗中执行“延期”或重建后更新环境变量。

注意：

- 脚本令牌是敏感信息，请勿写入代码仓库或对外传播。
- 令牌是账号级凭证，同账号下可复用到多个业务表脚本。

PowerShell 示例：

```powershell
$env:WPS_AIRSCRIPT_TOKEN="你的脚本令牌"
$env:WPS_WEBHOOK_MAP_PATH="d:\custome_development\wps多维表对接程序\wps_webhook_map.json"
```

### 2) 路由配置文件

编辑：`d:\custome_development\wps多维表对接程序\wps_webhook_map.json`

每个业务路由需要填写：

- `write_webhook`：录入脚本 webhook
- `query_webhook`：查询脚本 webhook
- `field_query_webhook`：字段配置查询脚本 webhook
- `default_notification_mode`：`text|image|file|dashboard_link`
- `default_range_mode`：`all|user|group|both`
- `range_filter_fields`：`user_field_name/group_field_name`

也可以使用交互式初始化脚本自动填写：

```powershell
python "d:\custome_development\wps多维表对接程序\wps_skill_init.py"
```

## 三、WPS 多维表端需要添加的通用脚本

每个需要接入的业务表都建议配置以下脚本：

1. 录入脚本：`录入脚本.js`
2. 查询脚本：`查询脚本.js`
3. 字段配置查询脚本：`字段配置查询脚本.js`

以上脚本在项目路径：

- `d:\custome_development\wps多维表对接程序\录入脚本.js`
- `d:\custome_development\wps多维表对接程序\查询脚本.js`
- `d:\custome_development\wps多维表对接程序\字段配置查询脚本.js`

### 3.1) 需要开启的 API 服务（非常重要）

在 WPS 脚本编辑器的“添加服务”里，建议按下列规则启用：

- 必开：`云文档API`
  - 原因：三份脚本都依赖 `Application.Sheet / Field / Record / View` 相关对象读写多维表。
- 建议开：`网络API`
  - 原因：当前三份脚本本身不强依赖外呼，但后续若脚本内加入外部HTTP请求（如回调、下载）会直接用到，提前开启可减少排障成本。
- 不必开：`邮件API`
  - 原因：当前方案没有发送邮件能力，不需要该服务。

最小可运行集合：`云文档API`。

### 3.2) 录入脚本会自动新增哪些系统字段

- 会自动创建：
  - `_请求ID`
  - `_提交人`
  - `_提交渠道`
- 已移除，不再自动创建：
  - `_提交来源`
  - `_原始提交记录`

说明：

- `_提交人` 与 `_提交渠道` 都是动态值，不是写死。
- `WPS_SUBMIT_CHANNEL` 可直接设置为你的指令来源渠道名，例如：`telegram`、`feishu`、`whatsapp`。

## 四、如何获取脚本 webhook

1. 打开目标多维表。
2. 进入自动化/脚本（AirScript）并新建脚本。
3. 粘贴脚本并保存发布。
4. 在脚本详情页复制 webhook。
5. 将 webhook 写入 `wps_webhook_map.json` 对应业务路由。

## 五、如何确认要操作的表名称

- 直接使用多维表页面顶部显示名称，填入路由的 `sheet_name`。
- 建议名称与实际保持完全一致（含大小写、空格、符号）。

## 六、Agent 调用说明

### 1) 首次调用建议先做健康检查

```powershell
$env:WPS_SKILL_MODE="setup"
python "d:\custome_development\wps多维表对接程序\wps_skill_router.py"
```

返回内容可检查：

- `token_configured`
- 每条路由的 `write_webhook_configured`
- 每条路由的 `query_webhook_configured`
- 每条路由的 `field_query_webhook_configured`

### 1.1) 交互式初始化（推荐）

```powershell
python "d:\custome_development\wps多维表对接程序\wps_skill_init.py"
```

该脚本会逐项询问：

- 路由key、业务名称、表名称
- 录入 webhook、查询 webhook、字段查询 webhook
- 默认通知模式、默认范围模式
- 范围筛选字段名映射

### 2) 获取某业务可填字段

```powershell
$env:WPS_SKILL_MODE="fields"
$env:WPS_SKILL_INTENT="报销"
python "d:\custome_development\wps多维表对接程序\wps_skill_router.py"
```

### 3) 录入数据

```powershell
$env:WPS_SKILL_MODE="create"
$env:WPS_SKILL_INTENT="报销"
$env:WPS_SKILL_DATA='{"日期":"2026-03-30","费用类型":"差旅费","报销金额":268.5}'
python "d:\custome_development\wps多维表对接程序\wps_skill_router.py"
```

### 4) 生成查询参数（给查询 webhook 用）

```powershell
$env:WPS_SKILL_MODE="query_argv"
$env:WPS_SKILL_INTENT="报销"
$env:WPS_QUERY_MONITOR_FIELD="报销状态"
$env:WPS_QUERY_MONITOR_CONTENT="待审核"
$env:WPS_QUERY_USER="张三"
$env:WPS_QUERY_GROUP="市场部"
python "d:\custome_development\wps多维表对接程序\wps_skill_router.py"
```

### 5) 直接执行查询（通过 query_webhook）

```powershell
$env:WPS_SKILL_MODE="query"
$env:WPS_SKILL_INTENT="报销"
$env:WPS_QUERY_MONITOR_FIELD="报销状态"
$env:WPS_QUERY_MONITOR_CONTENT="待审核"
$env:WPS_QUERY_USER="张三"
$env:WPS_QUERY_GROUP="市场部"
python "d:\custome_development\wps多维表对接程序\wps_skill_router.py"
```

### 6) 多条件查询（包含关系 + 数字范围）并按“记录N”文本输出

```powershell
$env:WPS_SKILL_MODE="query"
$env:WPS_SKILL_INTENT="报销"
$env:WPS_QUERY_MONITOR_FIELD="报销状态"
$env:WPS_QUERY_MONITOR_CONTENT="待审核"
$env:WPS_QUERY_RULE="等于"
$env:WPS_QUERY_RETURN_MODE="selected_fields"
$env:WPS_QUERY_RETURN_FIELDS="具体事由,报销金额,报销状态,支付方式"
$env:WPS_QUERY_CONDITIONS_JSON='[
  {"field":"具体事由","op":"contains","value":"附件"},
  {"field":"报销金额","op":"between","min":100,"max":170}
]'
$env:WPS_QUERY_AGG_JSON='{"op":"sum","field":"报销金额"}'
$env:WPS_QUERY_OUTPUT_FORMAT="text"
python "d:\custome_development\wps多维表对接程序\wps_skill_router.py"
```

可用操作符：

- 文本：`contains` / `not_contains`
- 数字：`gt` / `gte` / `lt` / `lte` / `between`

返回模式：

- `notification`：仅返回通知字段
- `selected_fields`：返回 `WPS_QUERY_RETURN_FIELDS` 指定字段
- `all_fields`：返回整条记录全部字段（默认排除附件字段）

聚合统计：

- `WPS_QUERY_AGG_JSON` 支持：`sum` / `avg` / `min` / `max` / `count`
- 示例：`{"op":"sum","field":"报销金额"}` 表示对筛选后的记录做报销金额求和

附件字段控制：

- `WPS_QUERY_INCLUDE_ATTACHMENTS`：`true|false`
- 默认 `false`，即“整条记录返回”时不带附件字段

## 七、推荐调用策略（给Agent）

1. 先 `setup`，确认 token 与 webhook 配置齐全。
2. 再 `fields`，拿到字段名与类型。
3. 引导用户补齐业务字段值。
4. 用 `create` 执行录入。
5. 若是查询场景，优先用机器枚举：
   - `range_mode`: `all|user|group|both`
   - `notification_mode`: `text|image|file|dashboard_link`
