```bash

```

将 `skills/wps-airscript-agent` 放入 Agent 的技能目录。

npx degit tengyuangangben/skills/wps-airscript-agent ./wps-airscript-agent

## 三、首次配置

### 1) 必需环境变量

- 无硬性必需（二选一）：
  - 在 `wps_webhook_map.json` 顶层配置 `token`
  - 或设置 `WPS_AIRSCRIPT_TOKEN`

### 2) 常用可选环境变量

- `WPS_WEBHOOK_MAP_PATH`
- `WPS_SUBMITTER`
- `WPS_SUBMIT_CHANNEL`
- `WPS_KEEP_AIRSCRIPT_LOGS`（默认 `false`，排障时可设 `true`）

若未设置 `WPS_WEBHOOK_MAP_PATH`，会自动按以下路径查找：

1. `scripts/wps_webhook_map.json`
2. Skill 根目录 `wps_webhook_map.json`
3. 当前工作目录 `wps_webhook_map.json`

PowerShell 示例：

```powershell
$env:WPS_AIRSCRIPT_TOKEN="你的脚本令牌"
$env:WPS_WEBHOOK_MAP_PATH="D:\path\to\wps_webhook_map.json"
$env:WPS_SUBMITTER="agent_user"
$env:WPS_SUBMIT_CHANNEL="telegram"
```

### 3) 路由配置文件

在 `wps_webhook_map.json` 为每个路由配置：

- 可选顶层：`token`（优先于环境变量 `WPS_AIRSCRIPT_TOKEN`）
- `write_webhook`
- `delete_webhook`
- `query_webhook`
- `field_query_webhook`
- `request_id_field_name`（可选，默认 `_请求ID`）
- `default_notification_mode`
- `default_range_mode`
- `range_filter_fields`

## 四、WPS 端脚本部署

每个业务表建议部署四份脚本并发布：

1. `scripts/录入脚本.js`
2. `scripts/查询脚本.js`
3. `scripts/字段配置查询脚本.js`
4. `scripts/删除脚本.js`

最小服务：`云文档API`。

## 五、字段写入约束（v1.1.2）

- 默认严格模式：仅按目标表已有字段写入
- 仅允许自动创建系统字段：
  - `_请求ID`
  - `_提交人`
  - `_提交渠道`
- 其余字段不存在时直接失败，不自动创建

仅当用户明确要求新增字段时才允许开启：

```powershell
$env:WPS_ALLOW_NEW_FIELDS="true"
$env:WPS_ALLOW_NEW_FIELDS_REQUESTED="true"
$env:WPS_NEW_FIELDS_WHITELIST="字段A,字段B"
```

说明：

- 未同时满足 `WPS_ALLOW_NEW_FIELDS=true` + `WPS_ALLOW_NEW_FIELDS_REQUESTED=true` + 白名单非空时，业务字段不会被自动创建。

## 六、提交人/提交渠道写入优先级

从高到低：

1. 显式参数（`submitter` / `submit_channel`）
2. OpenClaw/聊天运行时变量（如 `OPENCLAW_CHANNEL`、`OPENCLAW_CHAT_CHANNEL`、`REQUEST_CHANNEL`）
3. `WPS_SKILL_DATA` 元数据（`submitter`、`submit_channel`、`_提交人`、`_提交渠道`）
4. `WPS_SUBMITTER` / `WPS_SUBMIT_CHANNEL`
5. 默认值（`agent` / `wecom`）

OpenClaw 调用要求（强制）：

- 每次 `create/update/update_attachment/delete` 必须显式传 `submitter` 与 `submit_channel`。
- 若 `submit_channel` 缺失，不要直接提交，先从会话上下文补齐后再调用。
- 建议启用强校验：
  - `WPS_REQUIRE_SUBMIT_CHANNEL=true`
  - 建议同时开启：`WPS_REQUIRE_SUBMITTER=true`
  - `WPS_REQUIRE_CONFIRM_SUBMIT=true`
  - `WPS_FORBID_ATTACHMENT_OCR_BY_DEFAULT=true`

上述开关也可直接写入 `wps_webhook_map.json` 顶层 `config`：

- `require_submit_channel`
- `require_submitter`
- `require_confirm_submit`
- `forbid_attachment_ocr_by_default`

可选覆盖默认渠道：

- 全局：`WPS_FALLBACK_SUBMIT_CHANNEL`
- 路由级：`default_submit_channel`

## 七、常用调用命令

### 1) 健康检查

```powershell
$env:WPS_SKILL_MODE="setup"
python ".\scripts\wps_skill_router.py"
```

### 2) 字段发现

```powershell
$env:WPS_SKILL_MODE="fields"
$env:WPS_SKILL_INTENT="报销"
python ".\scripts\wps_skill_router.py"
```

`fields` 返回中会包含：

- `description`：字段说明（如“自动生成 / 唯一 / 必填 / 选项列表”）
- `options`：单选/多选可选项数组
- `required`、`is_primary`：校验约束信息

`create` 录入前会自动做预校验：

- 单选字段值不在 `options` 时会拦截并提示
- 多选字段存在非法值时会拦截并提示
- 新增模式下缺少必填字段会拦截并提示

### 3) 新增录入

```powershell
$env:WPS_SKILL_MODE="create"
$env:WPS_SKILL_INTENT="报销"
$env:WPS_SKILL_DATA='{"日期":"2026-03-30","费用类型":"差旅费","报销金额":268.5}'
python ".\scripts\wps_skill_router.py"
```

### 4) 补传附件到已有记录（推荐）

```powershell
$env:WPS_SKILL_MODE="update_attachment"
$env:WPS_SKILL_INTENT="花名册"
$env:WPS_UPDATE_KEY_FIELD="姓名"
$env:WPS_UPDATE_KEY_VALUE="李附件测试"
$env:WPS_UPDATE_ATTACHMENT_FIELD="相关资料附件"
$env:WPS_UPDATE_ATTACHMENT_MODE="append"
$env:WPS_UPDATE_ATTACHMENT='{"file_path":"C:\\Users\\sunli\\Downloads\\demo.pdf"}'
python ".\scripts\wps_skill_router.py"
```

### 5) 覆盖模式更新同一条记录（create方式）

```powershell
$env:WPS_SKILL_MODE="create"
$env:WPS_OVERWRITE_MODE="true"
$env:WPS_KEY_FIELD="姓名"
$env:WPS_KEY_VALUE="李附件测试"
$env:WPS_SKILL_DATA='{"相关资料附件":{"file_path":"C:\\Users\\sunli\\Downloads\\demo.pdf"}}'
python ".\scripts\wps_skill_router.py"
```

### 6) 查询（低 token 推荐）

```powershell
$env:WPS_SKILL_MODE="query"
$env:WPS_SKILL_INTENT="报销"
$env:WPS_QUERY_MONITOR_FIELD="报销状态"
$env:WPS_QUERY_MONITOR_CONTENT="待审核"
$env:WPS_QUERY_RULE="等于"
$env:WPS_QUERY_RETURN_MODE="selected_fields"
$env:WPS_QUERY_RETURN_FIELDS="具体事由,报销金额,报销状态"
$env:WPS_QUERY_OUTPUT_FORMAT="text"
python ".\scripts\wps_skill_router.py"
```

### 7) 删除记录（按条件/撤销）

```powershell
$env:WPS_SKILL_MODE="delete"
$env:WPS_SKILL_INTENT="花名册"
$env:WPS_DELETE_FIELD="姓名"
$env:WPS_DELETE_VALUE="李附件测试"
$env:WPS_DELETE_RULE="等于"
python ".\scripts\wps_skill_router.py"
```

按空值删除（推荐显式使用“为空”规则）：

```powershell
$env:WPS_SKILL_MODE="delete"
$env:WPS_SKILL_INTENT="花名册"
$env:WPS_DELETE_FIELD="客户姓名"
$env:WPS_DELETE_RULE="为空"
python ".\scripts\wps_skill_router.py"
```

按请求ID撤销：

```powershell
$env:WPS_SKILL_MODE="delete"
$env:WPS_SKILL_INTENT="花名册"
$env:WPS_DELETE_REQUEST_ID="employee_roster-ab12cd34"
python ".\scripts\wps_skill_router.py"
```

### 8) 更新已有记录字段（非附件）

```powershell
$env:WPS_SKILL_MODE="update"
$env:WPS_SKILL_INTENT="花名册"
$env:WPS_UPDATE_KEY_FIELD="姓名"
$env:WPS_UPDATE_KEY_VALUE="李附件测试"
$env:WPS_UPDATE_FIELDS_JSON='{"联系电话":"13800001111","客户状态":"活跃"}'
$env:WPS_UPDATE_MUST_EXIST="true"
python ".\scripts\wps_skill_router.py"
```

## 八、附件规范（避免灰色问号）

- 推荐优先：`file_path` / `file_url`
- 必要时可用：`file_data`（需配 `file_name`，带后缀）
- 支持格式：
  - `{"file_path":"D:\\a.pdf"}`
  - `{"file_url":"https://example.com/a.pdf","file_name":"a.pdf"}`
  - `{"files":[{"file_path":"D:\\a.pdf"},{"file_url":"https://..."}]}`

## 十、控制台中文乱码排查

- 路由脚本已内置 UTF-8 输出兼容处理（stdout/stderr）。
- 若宿主终端仍乱码，优先检查终端编码并确保 Python 运行环境可输出 UTF-8。
- 若仅 `field_query_webhook` 返回的 `raw_fields` 中文乱码而 `query_webhook` 正常，优先升级到最新 `scripts/wps_skill_router.py`（已增加响应字节级JSON解码兼容，避免Windows环境错误字符集推断）。
